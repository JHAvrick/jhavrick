# Self-Hosting A Chrome Extension

Did you know it's possible to self-host a Chrome extension? Don't get too excited, there are *a lot* of caveats. 

## Contents
[Prerequisites](#prerequisites) \
[Dependencies](#dependencies) \
[Sign and Pack the Extension](#sign-and-pack-the-extension)

## Why would you want to do this?

I can only concieve of two reasons you might want to self-host a Chrome extension:
 - You have developed an extension for use in an Enterprise organization and you don't want the extension published on the Chrome Web Store, even if unlisted
 - You want to deploy and test your extension in a production-like environment without publishing to the Chrome Web Store


## Prerequisites

 - [Google Workspace](https://admin.google.com/) (previously GSuite) Enterprise Account
 - A [device or browser enrolled in your organization](https://admin.google.com/ac/chrome/devices)
 - [NPM](https://www.npmjs.com/) / [Node](https://nodejs.org/en)

For the sake of brevity we're going to assume you already have a functioning Chrome Extension and the will to forge ahead.

## Dependencies

We're going to use two dependencies to make our life easier, though technically neither is an absolute necessity to get the job done.
 - [crx](https://www.npmjs.com/package/crx) - We'll use this to automate packing our extension as a CRX. You can also do this step manually each time using Chome if you desire.
 - [ngrok](https://www.npmjs.com/package/ngrok) - We'll use ngrok to temporarily host the extension. If you already have a more permanent hosting solution in mind, you can skip this.
```js
npm install --g crx ngrok
```

## Sign and Pack the Extension

#### Using CRX

If you've never packed the extension, simply navigate to your extension's root directory and run the following command:

`
crx pack -o
`

This command will also generate a `.pem` key file for you or use your existing key file if it is in the root directory. If you want to use a specific key file, you can specify it with the `-p` flag:


`
crx pack myFirstExtension -p myPrivateKey.pem -o
`

 Using the same `.pem` file to sign your extension each time is necessary to maintain the same extension ID, so keep track of this file. If you lose your dev `.pem` it may not be the end of the world, but you will need to redeploy the self-hosted extension.

 These examples are taken directly from the [crx](https://www.npmjs.com/package/crx) NPM guide.
